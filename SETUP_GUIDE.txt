#
#  STEP-BY-STEP SETUP GUIDE — JobHunter AI
#  Read this top to bottom. Takes ~30 minutes.
#
# ════════════════════════════════════════════════════════════════════════════

STEP 1 — INSTALL PYTHON
────────────────────────
Go to: https://www.python.org/downloads/
Download Python 3.11 or newer.
During install: ✅ CHECK "Add Python to PATH"
Verify: open Terminal/Command Prompt → type: python --version


STEP 2 — INSTALL GIT
──────────────────────
Go to: https://git-scm.com/downloads
Install with default settings.
Verify: git --version


STEP 3 — GET YOUR ANTHROPIC API KEY
─────────────────────────────────────
Go to: https://console.anthropic.com
Sign up / log in → API Keys → Create Key
Copy the key (starts with sk-ant-...)
Keep it safe — you'll need it in Step 6


STEP 4 — SET UP THE PROJECT FOLDER
────────────────────────────────────
Create a folder called "jobhunter" anywhere on your computer.
Inside it, create these files (copy the code from the files above):

  jobhunter/
  ├── main.py          (the FastAPI backend)
  ├── database.py      (database layer)
  ├── scraper.py       (job board scrapers)
  ├── applier.py       (auto-apply engine)
  ├── requirements.txt (dependencies)
  ├── Procfile         (Railway start command)
  ├── railway.json     (Railway config)
  └── frontend/        (copy your HTML files here)
       ├── index.html       (dashboard)
       ├── resume.html      (resume parser)
       ├── scraper.html     (job scraper)
       └── agent.html       (auto-apply engine)


STEP 5 — TEST LOCALLY (optional but recommended)
──────────────────────────────────────────────────
Open Terminal in your jobhunter folder:

  pip install -r requirements.txt
  playwright install chromium

Create a .env file with your keys (see .env template above).

Run locally:
  uvicorn main:app --reload

Open browser: http://localhost:8000
If it shows "JobHunter AI API" — it works!


STEP 6 — DEPLOY TO RAILWAY (free hosting)
───────────────────────────────────────────

a) Sign up at: https://railway.app (use GitHub login)

b) Push your code to GitHub:
   git init
   git add .
   git commit -m "JobHunter AI initial commit"
   (create a repo on github.com, then:)
   git remote add origin https://github.com/YOUR_USERNAME/jobhunter.git
   git push -u origin main

c) In Railway dashboard:
   → New Project → Deploy from GitHub Repo
   → Select your "jobhunter" repo
   → Railway auto-detects Python and starts building

d) Add PostgreSQL database:
   → In your Railway project → Add Service → Database → PostgreSQL
   → Railway creates a DB and gives you a DATABASE_URL automatically

e) Add environment variables:
   → Your service → Variables tab → Add:
     ANTHROPIC_API_KEY = sk-ant-your-key-here
     DATABASE_URL      = (Railway fills this automatically from the DB)

f) Railway will redeploy. In 2-3 minutes you'll get a URL like:
   https://jobhunter-production.up.railway.app

That's your live app URL. Share it with friends!


STEP 7 — CONNECT FRONTEND TO BACKEND
───────────────────────────────────────
In each HTML file, update the API base URL:

  const API = "https://your-app.up.railway.app/api";

Then all fetch() calls will hit the live backend.


STEP 8 — YOU'RE LIVE! 🎉
──────────────────────────
Open your Railway URL in browser.
Register an account.
Upload your resume.
Set preferences.
Hit "Start Scanning" — the agent will run every 3 hours automatically,
find jobs, match them to your profile, and auto-apply on your behalf.

You'll see every application logged in the Job Tracker.


TROUBLESHOOTING
────────────────
Problem: "Module not found"
Fix: Run: pip install -r requirements.txt

Problem: Playwright browser not found
Fix: Run: playwright install chromium

Problem: Database connection failed
Fix: Check DATABASE_URL in environment variables

Problem: API key error
Fix: Check ANTHROPIC_API_KEY is set correctly

Problem: Job board blocked the scraper
Fix: Some boards detect bots. We can add delays,
     rotate user agents, or use their official APIs.
     Tell me which board is failing and I'll fix it.


ADDING A NEW JOB BOARD (when you find one)
───────────────────────────────────────────
1. Open scraper.py
2. Copy any existing scraper function (e.g. scrape_simplyhired)
3. Rename it (e.g. scrape_newboard)
4. Update the CSS selectors to match the new board's HTML
5. Add it to the scrape_all_boards() task list
6. Push to GitHub → Railway redeploys automatically

Or just tell me the new board name and I'll write the scraper!


ADDING A NEW USER (friends)
────────────────────────────
Just share your Railway URL with them.
They register their own account, upload their own resume,
set their own preferences — the system handles everything separately per user.
